## 1. Product Vision & Architecture Design## 1.1 Executive Summary
NativeStream is a high-performance, private live sports streaming ecosystem designed as an empty, protocol-agnostic media player shell. To safely source dynamic, token-protected live sports streams for personal use without compromising public code distribution or App Store compliance, the system utilizes a Hard Architectural Boundary [1].
All volatile web-scraping and decryption operations are offloaded to an external, .gitignored Python script running on the local host machine. The core Go server discovery loop invokes this script asynchronously, receives a standardized JSON string payload over standard output (stdout), and seamlessly pipes the feeds through its native validation, health-monitoring, and reverse-proxy subsystems.

## 2. Functional Requirements (Go Core Backend)## 2.1 Ingestion Interface Integration (local_script.go)

* FR-1.1 Interface Conformance: The LocalScriptCrawler struct must seamlessly implement the existing Crawler interface required by discovery.NewEngine inside app/server/discovery/engine.go.
* FR-1.2 Boot Configuration: The module must dynamically check for the physical path of the local sidecar scraper script (e.g., ./crawlers/local_scraper.py or /etc/nativestream/plugins/scraper.py) based on settings loaded by config/config.go.
* FR-1.3 Safe Absence Fallback: If the external script file does not exist, the crawler must immediately return an empty data slice to e.fetchAll(ctx) and log a graceful debug message. The engine must proceed normally, presenting an empty, store-compliant distribution build.
* FR-1.4 Script Process Spawning: If the file is present, the module must utilize exec.CommandContext to trigger the local interpreter (e.g., python3) against the targeted script payload.

## 2.2 IPC Data Ingestion & Formatting

* FR-2.1 Stdout Pipeline Capture: The Go crawler must securely buffer and read the raw console output string (stdout) generated by the child process execution.
* FR-2.2 JSON Unmarshaling: The backend must parse the captured JSON bytes directly into standard internal target schemas ([]CandidateLink).
* FR-2.3 Pipeline Hand-Off: Discovered channel elements must automatically proceed downstream to matcher.Match, validator.Submit for ongoing uptime verification, and eventually save to store/store.go.

## 2.3 Transparent HLS Reverse Proxying

* FR-3.1 Dynamic Token Masking: When generating the output via playlist.Generate(), the server must strip out dangerous external URLs and replace them with local loops: http://localhost:8888/stream/{id}/proxy.
* FR-3.2 Header Spoofing Injection: When the native players hit the proxy, proxy/proxy.go must dynamically inject the target custom User-Agent and Referer values harvested by the script on the fly to bypass anti-hotlinking blocks.

## 3. Script Implementation & Decryption Logic (cricfy_scraper.py)
The independent Python script operates within an isolated sandbox. It performs the sequence below to parse streams without drawing any dependencies from heavy graphical frontends or frameworks like Kodi.
## 3.1 Three-Stage Execution Sequence

   1. Directory Ingestion (Category Sorting): The script executes an initial HTTP payload fetch against the root provider configuration mirror. It strips out indices, maps metadata, and converts raw groupings (e.g., Live Cricket, Football) into clean string fields.
   2. Key Resolution & AES Decryption: The script ports the algorithmic decryption keys directly from open-source repository assets (such as the Cricfy plugin's crypto_utils.py). It handles incoming scrambled response tokens, processes them using lightweight crypto libraries (e.g., pycryptodome), and extracts the true, tokenized underlying .m3u8 playlist destination.
   3. Payload Finalization: The script formats the data elements, sets structural network headers required by the source platform, and prints the raw JSON output buffer straight to the console using a standard print(json.dumps(channels)) command.

## 4.1 Go Crawler Module (app/server/discovery/crawlers/local_script.go)

package crawlers
import (
	"context"
	"encoding/json"
	"fmt"
	"os"
	"os/exec"
	"time"
)
// CandidateLink maps directly to the structure expected by discovery/engine.gotype CandidateLink struct {
	URL         string            `json:"url"`
	ChannelName string            `json:"channel_name"`
	GroupTitle  string            `json:"group_title"`
	TvgID       string            `json:"tvg_id"`
	LogoURL     string            `json:"logo_url"`
	Headers     map[string]string `json:"headers"`
	ChannelID   string            `json:"-"`
	SourceURL   string            `json:"-"`
}
type LocalScriptCrawler struct {
	ScriptPath string
}
func NewLocalScriptCrawler(path string) *LocalScriptCrawler {
	return &LocalScriptCrawler{ScriptPath: path}
}
func (c *LocalScriptCrawler) Name() string {
	return "local-script-crawler"
}
// Fetch matches the entry logic invoked during engine.fetchAll(ctx)func (c *LocalScriptCrawler) Fetch(ctx context.Context) ([]CandidateLink, error) {
	// Verify file presence before executing shell environment
	if _, err := os.Stat(c.ScriptPath); os.IsNotExist(err) {
		return nil, nil // Return gracefully to maintain App Store shell compliance
	}

	// Protect the core process pool from hanging script executions
	execCtx, cancel := context.WithTimeout(ctx, 15*time.Second)
	defer cancel()

	cmd := exec.CommandContext(execCtx, "python3", c.ScriptPath)
	outputBytes, err := cmd.Output()
	if err != nil {
		return nil, fmt.Errorf("failed executing local script scraper: %w", err)
	}

	var candidates []CandidateLink
	if err := json.Unmarshal(outputBytes, &candidates); err != nil {
		return nil, fmt.Errorf("failed decoding local scraper JSON stdout payload: %w", err)
	}

	return candidates, nil
}

## 5. Non-Functional Requirements & Performance Enforcements

## 5.1 Fault Isolation & Thread Protection

* NFR-5.1 Execution Deadlines: The exec.CommandContext call must strictly enforce an upper-bound process execution limit of 15 seconds. If the third-party web scraper script hangs or falls victim to remote server performance locks, the Go core must forcibly close the child thread to protect server availability.
* NFR-5.2 Asynchronous Memory Streaming: When client applications stream video fragments through proxy/proxy.go, the proxy must pipe media bytes continuously using non-blocking buffers (io.Copy). This guarantees that volatile streams do not leak RAM or block the core HTTP server routing processes.

## 5.3 App Store Review Compliance

* NFR-5.3 Repository Sanitization: The filename of the personal python script (cricfy_scraper.py or equivalent configuration) must be explicitly listed inside the project's root .gitignore block. No proprietary scraping signatures or trademark definitions can exist in the compiled source code.
* NFR-5.4 Sandboxed Client Safety: Native client loops on Android and macOS must restrict multimedia network handshakes to independent background threads. If the Go proxy suffers upstream packet drops or link death, client frontends must elegantly show a native buffer indicator or a clean error message, keeping UI frames processing smoothly at 60fps.

## 6. Verification & Acceptance Criteria

* AC-1: Building and launching the NativeStream project with no external script file present generates a clean, compliant, empty-shell player with zero compilation or runtime faults.
* AC-2: Placing the active Python scraper on your server machine or mounting it into your distributed container stack via a docker-compose.yml volume instantly triggers ingestion during the Go engine's background runCycle.
* AC-3: If the remote streaming site introduces breaking network updates, the Go backend process and application clients remain untouched. Amending the standalone Python script file resolves the service failure seamlessly without needing to redeploy or recompile the core Go server or client applications.
* AC-4: The client application UI interfaces successfully display independent channel sorting options mapped entirely from the parsed script tags via the dynamic local GET /playlist.m3u route.
